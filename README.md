# Сайт Александра Белякова — техническая документация

Этот файл — справка для владельца сайта (не программиста) и для новой сессии Claude Code, чтобы быстро понять структуру и историю проекта без необходимости пересматривать всю переписку.

## 1. Общее описание

- **Что это:** лендинг частного специалиста по Яндекс Директу (Александр Беляков) + раздел бесплатных обучающих материалов (гайдов).
- **Домен:** `direct.alex-belyakov.com` (задаётся файлом `CNAME` в корне репозитория).
- **Репозиторий:** https://github.com/Aleksandr-Beliakov/direct
- **Хостинг:** GitHub Pages, статический сайт без сборки — чистый HTML/CSS/JS, без npm/фреймворков.
- **Шрифт:** Google Fonts «Golos Text», подключается на каждой странице через `<link>` в `<head>`.
- **Стиль вёрстки:** большая часть оформления — инлайн `style="..."` прямо в HTML (так исторически сложилось при переносе из макета Claude Design). В `css/style.css` вынесены только адаптивные media-запросы и несколько именованных компонентов (FAQ-аккордеон, попапы).

## 3. Все формы на сайте

На сайте 6 разных форм. Пять из них ведут на `thanks.html`, одна (подписка на гайды) — на `thanks-materials.html`.

| Название (внутреннее) | Где на сайте | Поля (name → что значит) |
|---|---|---|
| **Форма внизу главной** (`id="form-request"`, секция `#request`) | Финальный тёмный блок в самом низу главной страницы | `name` — имя клиента; `messenger` — мессенджер/телефон для связи; `link` — ссылка на сайт/рекламный аккаунт; `comment` — комментарий (необязательное поле) |
| **forma-audit** (`id="popup-audit"`) | Попап «Получите бесплатный разбор рекламы». Открывается с кнопок «Получить аудит» (шапка), «Получить бесплатный аудит» (хиро, карточка боли №2, блок бесплатного аудита, кейсы «Хочу также»). Также подключена отдельной копией на всех 3 страницах гайдов в `/materials/` (там кнопка «Получить бесплатный аудит» в финальном блоке статьи) | `phone` — телефон или мессенджер |
| **forma-obsudit** (`id="popup-obsudit"`) | Попап «Обсудить проект». Открывается с кнопок «Обсудить проект» (хиро, карточка боли №3) и «Обсудить ведение» (тариф «Месячное ведение») | `phone` — телефон или мессенджер |
| **forma-zakazat** (`id="popup-zakazat"`) | Попап «Заказать настройку». Открывается с кнопки «Заказать настройку» (тариф «Разовый запуск») | `phone` — телефон или мессенджер |
| **forma-mediaplan** (`id="popup-mediaplan"`) | Попап «Получить медиаплан». Открывается с кнопки «Получить бесплатный медиаплан» (карточка боли №1) | `phone` — телефон или мессенджер |
| **forma-gaidy** (`id="popup-gaidy"`) | Попап «Оставьте Email, чтобы открыть материалы». Открывается с трёх кнопок «Скачать бесплатно» в блоке «Бесплатные материалы» на главной. **Другая логика редиректа** — см. раздел 4 | `email` — почта |

Общие для всех форм технические поля:
- `consent` — чекбокс согласия на обработку персональных данных. **Обязателен**: без галочки форма не отправится, рядом появится подсказка.
- `subject` (скрытое поле) — пишет в письмо, какая именно форма/страница прислала заявку, например «Заявка с сайта — попап «Заказать настройку»». Без этого поля все письма выглядели бы одинаково.
- `botcheck` (скрытое поле-ловушка для ботов, honeypot) — см. раздел 4.

Три текстовых ссылки «Или скачайте: …» под карточками боли на главной — это **не формы**, а обычные якорные ссылки на блок `#materials» (см. раздел 5).

## 4. Как устроена отправка форм

- **Сервис:** [Web3Forms](https://web3forms.com) — сторонний сервис приёма форм без бэкенда, письмо с содержимым формы приходит на почту владельца ключа. Общий для всех форм access key хранится прямо в `js/forms.js` (не в этом файле — ключ намеренно не публикуется в документации).
- **Как работает `js/forms.js`:** один общий скрипт находит на странице все `<form data-redirect="...">`, при отправке:
  1. Проверяет чекбокс `consent`. Если не отмечен — форма не отправляется, показывается подсказка рядом с галочкой.
  2. Собирает все реально существующие поля формы через `FormData` и отправляет POST-запросом на `https://api.web3forms.com/submit`.
  3. Если Web3Forms ответил успехом — браузер переходит на адрес из `data-redirect` (`thanks.html` или `thanks-materials.html`, для страниц в `/materials/` — с префиксом `../`).
  4. Если сеть недоступна или Web3Forms вернул ошибку — кнопка возвращается в исходное состояние, рядом с формой показывается текст «Не получилось отправить, напишите мне напрямую в Telegram» со ссылкой.
- **Поле `subject`** — задаёт «тему» письма на стороне Web3Forms, по нему в почте сразу видно, с какой именно кнопки/попапа пришла заявка.
- **Honeypot (`botcheck`)** — скрытый чекбокс, невидимый обычному человеку (`display:none`), но который боты-скрипты иногда всё равно заполняют. Web3Forms распознаёт такие отправки как спам и отсекает их. Реализовано на всех формах сайта.
- **Куда приходит почта:** на адрес, привязанный к access key в личном кабинете Web3Forms (настраивается на их стороне, в коде сайта этого адреса нет).

## 7. Внешние сервисы

- **Web3Forms** — приём и пересылка данных всех форм сайта на почту владельца. Никакого бэкенда/базы данных на сайте нет, вся обработка форм происходит на стороне Web3Forms. Access key используется единый для всех форм, хранится в `js/forms.js` (сам ключ в этой документации не публикуется).
- **GitHub Pages** — хостинг статического сайта. Домен `direct.alex-belyakov.com` подключён через файл `CNAME` в корне репозитория + настройку custom domain в настройках репозитория на GitHub (там же выпускается HTTPS-сертификат).
- **Google Fonts** — подключение шрифта Golos Text через `<link>` на каждой странице (CDN Google, не требует настройки).
- **Яндекс.Метрика** — счётчик подключён на всех страницах сайта (id счётчика `109844122`), код вставлен в самое начало `<head>` на каждой странице, как рекомендует Яндекс. **При создании любой новой страницы сайта счётчик нужно добавлять туда же, в начало `<head>`** — это правило действует на все будущие страницы, не только на уже существующие.
